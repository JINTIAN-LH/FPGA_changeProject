# 任务与分工总表（小白执行版）

## 1. 目标

让每位成员知道“自己今天改什么、怎么验收、交付什么”。

## 2. 分工总览

| 角色 | 负责模块 | 每日输出 |
|---|---|---|
| 组员1 | 数据采集与清洗 | 可复现的数据样本 |
| 组员2 | 上位机协议与 UDP | 可跑的收发脚本 |
| 组员3 | FPGA 工程与时钟/约束 | 可综合工程 |
| 组员4 | FPGA 协议解析 | 解析模块 + TB |
| 组员5 | 指标硬件化 | 指标模块 + 对比报告 |
| 组员6 | 决策与回包 | 回包模块 + 用例 |
| 组员7 | 顶层整合与上板 | 可运行 bit 与联调记录 |
| 组员8 | 仿真、可视化、文档 | 测试报告与演示材料 |

## 3. 每人统一工作流

1. 拉最新代码
2. 只改自己模块
3. 先跑本地测试
4. 更新文档
5. 更新 commit.md

## 4. 验收口径（统一）

### 上位机侧

- Python 回归通过
- 异常注入通过

### FPGA侧

- batch 仿真通过
- 关键信号可观察

## 5. 新手常见误区

1. 只改代码不改文档
2. 跳过测试直接提交
3. 修改协议但不通知上下游

## 6. 每日站会模板（推荐）

1. 昨天完成了什么
2. 今天准备做什么
3. 是否卡点（需要谁配合）
4. 是否影响本周里程碑
