# FPGA 高速交易系统 — 数据流全图 (小白也能懂)

> 读完这篇文档，你会知道：**数据从股市行情网站出发 → 穿过网线到 FPGA → 算出一堆指标 → 再回传到电脑** 的整个过程。

## 1. 用大白话说：每一步在干什么

### 第 0 步：电脑采集行情

```
新浪财经 API ──→ Python easyquotation ──→ 每 3 秒拉一次最新股价
                                              │
                                    ┌─────────┴─────────┐
                                    │ 滑动窗口保存最近 20 条│
                                    │ (相当于 1 分钟的数据) │
                                    └─────────┬─────────┘
                                              │
                              每分钟整点时，挤出这一分钟的数据：
                              O=第一笔价格  H=最高价  L=最低价
                              C=最后一笔   V=累计成交量
                                              │
                                    打包成 48 字节帧
```

### 第 1 站：以太网 PHY (RTL8211F)

> 把网线上的电信号翻译成数字信号，反之亦然。

- 电脑通过网线发过来的高低电平 → PHY 芯片转换成 RGMII 格式的并行数据
- RGMII = 4 根数据线 + 1 根时钟线 (125MHz)，每时钟传 4 位
- PHY 地址由 MDIO 总线管理 (eth_mdio / eth_mdc)

### 第 2 站：以太网桥接 (board_eth_bridge)

> 可以理解为 FPGA 内部的「网卡驱动」。

| 子模块 | 功能 | 比喻 |
|--------|------|------|
| `mac_rx` | 解析以太网帧头，过滤 MAC 地址 | 邮局分拣：只看寄给我的信 |
| `ip_udp_parser` | 解析 IP/UDP 头，过滤端口号 | 只看 5001 端口的包裹 |
| `rgmii_rx_sync` | RGMII 双沿采样 → 单沿对齐 | 翻译不同「口音」的数据 |
| `udp_tx_engine` | 把结果组包成 UDP 帧发出 | 把回信装进信封 |
| `mdio_init_fsm` | 上电初始化 PHY 寄存器 | 开机自检 |

### 第 3 站：行情帧解码器 (market_frame_decoder)

> 把 48 个字节拆成有用的数字。

```
48 字节的原始数据 = [帧头 2B][长度 2B][代码 8B][时间 4B][O 4B][H 4B][L 4B][C 4B][V 4B][保留 8B][CRC 4B]
                       ↓
                open=10.23  high=10.56  low=10.12  close=10.45  volume=1234567
```

**CRC32 校验**：就像快递单号，如果传输过程中有字节错误，CRC 对不上，这帧就被丢弃。

### 第 4 站：行情注入 (board_market_feed_ingress)

> 做两件事：对齐时序 + 解析配置字节。

配置字节说明：
- `cfg_strong_buy` = 强买阈值 (0-100)
- `cfg_buy` = 买入阈值
- `cfg_neutral` = 中性阈值
- `cfg_sell` = 卖出阈值

这些配置目前固化在 FPGA 内，后续版本可以通过 UDP 动态配置。

### 第 5 站：指标计算引擎 (indicator_top) ★ 核心 ★

> 这是 FPGA 真正的价值所在 —— **8 个指标同时并行计算，延迟固定 1-2 个时钟周期 (20-40ns)**。

| 指标 | 算法 | 一句话解释 | 计算模块 |
|------|------|-----------|----------|
| **MA5/20/60** | 简单移动平均 | 最近 N 根 K 线的平均收盘价 | `ma_calc` (运行和法) |
| **RSI** | 相对强弱指数 | 涨的力量 vs 跌的力量 (0-100) | `rsi_calc` (流水线+阈值) |
| **MACD** | 指数平滑异同 | 快线-慢线, 看趋势转折 | `macd_calc` (2级流水线) |
| **Bollinger** | 布林带 | MA20 ± 偏移量, 看波动范围 | `indicator_top` 内联 |
| **ATR** | 平均真实波幅 | 最高-最低, 看日内振幅 | `indicator_top` 内联 |
| **Vol Ratio** | 量比 | 当前成交量 vs 近期平均 | `vol_ratio_calc` (阈值比较) |

**FPGA vs CPU 的关键差异**：
- CPU：8 个指标串行计算，一个完了再算下一个，延迟 = 8× 单指标时间
- FPGA：8 个指标同时启动，延迟 = 最慢一个的时间 (所有结果同时出炉)

### 第 6 站：协议组包 (m1_protocol_core)

> 把算好的指标值打包成 44 字节的下行帧。

同时验证上游帧的 CRC32：
- ✅ CRC 正确 → `frame_accepted` = 1, 发送回包
- ❌ CRC 错误 → `frame_rejected` = 1, `reject_reason` = REJ_CRC (0x3), 不发回包

### 第 7 站：UDP 回传

> FPGA 把 44 字节装进 UDP 包 → 通过 PHY → 网线 → 电脑。

回包地址硬编码在 FPGA 中：
- 目标 IP: `192.168.100.104` (你的电脑)
- 目标端口: `5000`
- 源 IP: `169.254.0.118` (FPGA 自身)

## 3. 怎么跑起来

### 前置条件

| 条件 | 检查方法 |
|------|----------|
| FPGA 已烧录 | Vivado Hardware Manager 显示 `Programmed` |
| 网线直连 PC ↔ FPGA | 以太网指示灯亮 |
| PC 以太网 IP = 192.168.100.104 | `ipconfig` 查看 (如不是，需管理员权限修改) |
| Python 依赖已安装 | `pip install akshare easyquotation requests` |

### 网络配置 (一次性设置)

**如果 PC 以太网接口没有 192.168.100.104**，以管理员身份运行：

```powershell
# 方法 1: 添加辅助 IP (推荐，不影响原有 DHCP)
netsh interface ip add address "以太网" 192.168.100.104 255.255.255.0

# 方法 2: 直接改静态 IP
netsh interface ip set address "以太网" static 192.168.100.104 255.255.255.0
```

### 启动全栈

```bash
cd d:\FPGADevelopMent\fpga_exchangeSerdes

# 完整模式: 实时采集 + FPGA 指标计算
python host_side/app/run_all.py 000858

# 烟雾测试: 发一帧合成数据，验证链路
python host_side/app/e2e_runner.py --code 000858 --smoke

# 离线回放: 用历史 JSON 文件发给 FPGA
python host_side/app/e2e_runner.py --code 000858 --limit 30

# 纯软件模式 (不需要 FPGA): 启动 mock FPGA 服务器
python host_side/app/e2e_runner.py --code 000858 --start-mock --limit 30
```

### 输出文件

| 文件 | 说明 |
|------|------|
| `{code}_daily_minute.json` | 历史分钟 K 线存档 |
| `{code}_real_time_window.json` | 实时滑动窗口 (最近 20 条 3 秒采样) |
| `{code}_fpga_responses.json` | FPGA 返回的所有指标结果 |
| `fpga_link_events.jsonl` | UDP 链路日志 (每帧的 RTT/重试/CRC 状态) |

## 5. FPGA 内部时钟域

```
           ┌─────────────────────────────┐
           │   sys_clk_50m  (50 MHz)     │ ← 板载晶振, 几乎所有逻辑

           │   indicator / protocol /    │
           │   frame_decoder / ingress   │
           └──────────────┬──────────────┘
                          │
```

## 6. 常见问题排查

| 症状 | 可能原因 | 解决方法 |
|------|----------|----------|
| `ping 169.254.0.118` 不通 | 网线未连/PHY 未初始化 | 检查网线, 等 FPGA 上电 2 秒后重试 |
| 发送成功但无回包 | PC IP 不是 192.168.100.104 | 用 `netsh` 配置以太网 IP |
| CRC error 很多 | 网线质量差/PHY 配置错误 | 换网线, 检查 PHY 自协商状态 |
| `ModuleNotFoundError: akshare` | 缺少 Python 依赖 | `pip install akshare easyquotation requests` |
| FPGA 收不到包 | 端口 5001 被占用 | 检查防火墙, 关闭其他占用 5001 端口的程序 |
